# Льготное питание колледжа — Frontend

Frontend работает в backend-first режиме.

## Локальный запуск

```bash
npm install
npm run dev
```

`npm run dev` поднимает frontend только локально на текущем компьютере.

Для доступа из локальной сети колледжа используйте:

```bash
npm run dev:lan
```

Для подготовки установленного кассового PWA и проверки оффлайн-режима смотрите:

- [../docs/cashier-pwa-offline.md](../docs/cashier-pwa-offline.md)

По умолчанию:

- frontend ходит на `'/api'`
- Vite proxy направляет `'/api'` на `http://localhost:5000`
- mock выключен

Перед запуском frontend рекомендуется поднять backend локально:

```bash
cd ../backend
python run.py
```

Backend в этом сценарии можно оставить локальным на `127.0.0.1:5000`, потому что запросы второго пользователя пойдут через Vite proxy на той же машине.

Чтобы быстро узнать адрес, который должен открыть другой человек в сети колледжа, из корня репозитория выполните:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\show-lan-url.ps1
```

## Env

Создайте `.env.local` на основе `.env.example`, если нужно переопределить локальные значения.

Стандартный backend-first набор:

```env
VITE_DEV_HOST=127.0.0.1
VITE_DEV_PORT=5173
VITE_PROXY_TARGET=http://localhost:5000
VITE_API_URL=/api
VITE_USE_MOCK_API=false
```

## Политика по mock

- backend — единственный source of truth
- mock — только узкий dev/demo fallback
- в mock нельзя добавлять новую доменную логику
- новые фичи идут через backend first
- если backend-контракт изменился, mock нужно обновить в том же изменении или убрать для этого сценария
- mock не должен скрывать реальные `4xx/5xx`

## Когда mock допустим

- изолированная UI-разработка
- короткие демо
- временная недоступность backend в dev

Mock нельзя использовать как замену проверке реальной бизнес-логики.

## Кассовое PWA

Кассовый оффлайн-сценарий поддерживается только после онлайн-подготовки в том же браузере и на том же origin.

Коротко:

1. открыть приложение по рабочему адресу, например `http://172.20.100.168:5173`;
2. войти под кассиром;
3. открыть `/cashier` и дать приложению время на фоновый прогрев кассовых экранов;
4. установить PWA;
5. проверить запуск без интернета.

Подробная пошаговая инструкция:

- [../docs/cashier-pwa-offline.md](../docs/cashier-pwa-offline.md)
