# PWA кассы и оффлайн-работа

Эта инструкция описывает, как подготовить кассовое рабочее место к работе без интернета через установленное PWA-приложение.

Документ рассчитан на сценарий, когда касса открывается с адреса вида `http://<IP-адрес>:5173` в локальной сети колледжа.

## Что поддерживается оффлайн

После корректной онлайн-подготовки в установленном PWA должны открываться:

- `/cashier`
- `/cashier/terminal`
- `/cashier/journal`

В оффлайн-режиме касса использует:

- локально сохранённый оффлайн-снимок студентов, талонов и категорий;
- локальную очередь выдач;
- локальный журнал событий этого терминала.

## Что не работает без сети

Без интернета нельзя рассчитывать на:

- первый вход в систему;
- первую подготовку терминала на новом устройстве;
- серверную сводку `/cashier/summary`;
- работу с ролями кроме кассира;
- обновление оффлайн-снимка, если он ещё не был подготовлен.

## Важные ограничения

- PWA привязано к точному origin. Если приложение установили с `http://172.20.100.168:5173`, потом нужно открывать его с этого же адреса.
- Другой IP, другой hostname или другой браузерный профиль считаются другим приложением и другим кэшем.
- После очистки browser storage, переустановки браузера или смены устройства онлайн-подготовку нужно выполнить заново.

## Подготовка терминала к оффлайн-работе

### 1. Поднимите frontend в сети

Для локальной сети колледжа:

```bash
cd frontend
npm run dev:lan
```

Или используйте ваш текущий рабочий контур, если он уже раздаёт frontend в сеть.

### 2. Откройте приложение с будущего кассового устройства

На кассовом устройстве откройте именно тот адрес, с которого приложение потом будет использоваться, например:

```text
http://172.20.100.168:5173
```

Не подготавливайте PWA на `localhost`, если потом кассир будет работать по сетевому IP.

### 3. Войдите под кассиром при наличии интернета

Нужен реальный онлайн-вход. Без него PWA не сможет восстановить кассовую сессию и локальный оффлайн-контекст.

### 4. Откройте меню кассы и дождитесь подготовки локальных данных

Перейдите в `/cashier`.

На экране кассы должно завершиться:

- подготовка оффлайн-данных;
- привязка терминала;
- получение оффлайн-гранта;
- сохранение оффлайн-снимка.

Практический ориентир:

- в меню кассы видно статус `Оффлайн-данные готовы`;
- нет баннера о том, что оффлайн-данные отсутствуют;
- терминал открывается при включённом интернете.

### 5. Дождитесь автоматического прогрева кассовых экранов

После онлайн-открытия `/` или `/cashier` приложение само прогревает критичные кассовые экраны:

- `/cashier`
- `/cashier/terminal`
- `/cashier/journal`

Ручное открытие каждой из этих страниц больше не является обязательным шагом.

Практически это означает:

- достаточно открыть кассовое меню онлайн;
- подождать несколько секунд, пока приложение закончит фоновую подгрузку;
- после этого можно устанавливать PWA и проверять оффлайн.

Если хотите сделать дополнительную проверку перед сменой, можно вручную один раз открыть `/cashier/terminal`, но это уже smoke-check, а не обязательная подготовка.

### 6. Установите PWA

В Edge или Chrome:

1. откройте меню браузера;
2. выберите установку приложения;
3. подтвердите установку.

После установки запускайте кассу именно как отдельное приложение, а не как обычную вкладку.

### 7. Проверьте оффлайн до реальной смены

Рекомендуемая проверка:

1. закройте обычную browser-вкладку;
2. запустите установленное приложение;
3. отключите интернет;
4. снова откройте PWA;
5. проверьте, что открывается `/cashier`, и при необходимости выполните smoke-check `/cashier/terminal`.

Если терминал не открывается оффлайн, устройство ещё не подготовлено до конца.

## Как работать кассиру через PWA

### Обычный порядок работы

1. Откройте установленное приложение.
2. Если интернет есть, дождитесь, пока касса догрузит актуальные данные.
3. Перейдите в `Терминал`.
4. Работайте как обычно.

### Если интернет пропал во время смены

1. Не переустанавливайте приложение.
2. Оставайтесь в установленном PWA.
3. Работайте через `/cashier/terminal`.
4. Оффлайн-выдачи будут складываться в локальную очередь этого терминала.
5. После восстановления связи очередь отправится автоматически.

## Что касса должна показывать оффлайн

При корректной подготовке оффлайн:

- меню кассы открывается;
- терминал открывается;
- журнал кассы открывается;
- summary может быть недоступна;
- по чужому корпусу касса должна показать запрет, а не `Талон не найден`;
- если этот же терминал уже выдал питание сегодня, ложного `Выдайте питание` быть не должно.

## После обновления frontend

Если frontend обновили:

1. подключите устройство к интернету;
2. откройте установленное PWA;
3. дайте приложению обновиться;
4. снова откройте `/cashier` и дайте приложению закончить фоновый прогрев кассовых экранов;
5. только потом проверяйте оффлайн повторно.

## Если оффлайн-терминал не открывается

Проверьте по порядку:

1. Устройство входило в систему под кассиром онлайн?
2. Используется тот же адрес, с которого PWA было установлено?
3. Открывалось ли кассовое меню `/cashier` онлайн?
4. Появлялся ли статус `Оффлайн-данные готовы`?
5. Успело ли приложение побыть онлайн несколько секунд после открытия кассового меню?
6. Не были ли очищены данные браузера?
7. Не обновился ли frontend без повторного онлайн-запуска PWA?

Если хотя бы один ответ отрицательный, повторите подготовку заново при наличии интернета.

## Рекомендуемый чек-лист перед выдачей

- устройство открывает установленное PWA;
- кассир вошёл под своей учётной записью;
- в `/cashier` видно, что оффлайн-данные готовы;
- `/cashier/terminal` открывается хотя бы в рамках smoke-check;
- `/cashier/journal` открывается;
- пробный оффлайн-запуск был выполнен заранее.
